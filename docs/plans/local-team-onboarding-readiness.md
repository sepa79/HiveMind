# Local Team Onboarding Readiness Plan

Goal: make HiveMind easy for a teammate to clone, start locally, inspect, and connect to an MCP client without needing project-author context.

## Scope

- Keep this local-first.
- Do not add auth, hosted multi-user deployment, or production packaging yet.
- Prefer explicit scripts and docs over hidden automation.
- Keep HiveMind as project memory/audit tooling, not task management software.

## Checklist

- [x] Add a team quickstart focused on the first 10 minutes.
- [x] Add MCP client configuration examples.
- [x] Add API health/status diagnostics.
- [x] Add a local doctor/smoke command.
- [x] Add a generic project init command for other local repos.
- [x] Document local data-root isolation and reset.
- [x] Verify docs, doctor, API, UI, and tests.

## Notes

- Feedback entries should link this plan with a `repo_file` reference so later agents can continue from the same checklist.
- Progress entries should record meaningful milestones, not every checkbox edit.
- If a checkbox is completed, a short progress entry is useful only when it helps the next contributor understand what changed.
